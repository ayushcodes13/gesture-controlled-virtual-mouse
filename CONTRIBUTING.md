# Contributing to AI Virtual Mouse

Thank you for your interest in contributing to the AI Virtual Mouse project! We welcome contributions from everyone, whether you're fixing a bug, improving documentation, or adding new features.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please search the existing issues to see if it has already been reported. If not, open a new issue and include:
- A clear, descriptive title.
- Steps to reproduce the bug.
- Your operating system and Python version.
- Any error messages or relevant logs.

### 2. Suggesting Enhancements
We're always looking for ways to improve! If you have an idea for a new feature or improvement:
- Open an issue with the "enhancement" label.
- Describe the feature as clearly as possible.
- Explain why this enhancement would be useful.

### 3. Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bugfix (`git checkout -b feature/your-feature-name`).
3. Make your changes and ensure they follow the existing code style.
4. Commit your changes with clear, descriptive commit messages.
5. Push your branch to your forked repository.
6. Open a Pull Request against the `main` branch of the original repository.

## Development Setup
Follow the [Quick Start](README.md#quick-start) guide in the README to set up your development environment.

## Code Style
Please follow [PEP 8](https://pep8.org/) for Python code. Ensure your code is well-commented and includes type hints where appropriate.
